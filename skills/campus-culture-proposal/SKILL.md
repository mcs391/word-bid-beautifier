# 全案型方案 HTML 生成器

## 用途

将任何类型的项目提案、服务方案、建设规划等以**单文件 HTML** 形式输出，可直接在浏览器打开、通过微信/邮件发送，适用于客户汇报、招标提案、内部评审等场景。

本 Skill 因校园文化全案项目实践沉淀而来，但其设计理念和模板结构**适用于任何行业/类型的方案文档**。

## 使用时机

用户描述以下意图时触发本 Skill：
- "帮我做一个 XX 方案 HTML"
- "生成项目提案网页"
- "制作服务方案"
- "做一个汇报材料"
- "XX 全案建设规划"

## 核心资产

模板文件位于 `assets/proposal_template.html`，使用 `{{PLACEHOLDER}}` 格式标记可替换内容。所有占位符均按实际项目替换，不使用默认值时该字段留空或显示"待定"。

## 工作流程

### Step 1：收集项目信息

向用户确认以下关键信息：

| 信息项 | 说明 |
|--------|------|
| 方案名称 | 完整标题，如"XX公司数字化转型方案" |
| 方案类型 | 建设规划/服务提案/产品方案/战略规划等 |
| 预算范围 | 如"30-50万"、"待定" |
| 建设周期 | 如"3-6个月"、"分三期" |
| 客户/受众 | 发送给谁看，影响语言风格 |
| 核心诉求 | 用户最想传达的 1-3 个重点 |
| 特殊要求 | 用户额外说明的内容 |

> **注意**：用户只提供名称和基本描述时，直接使用通用模板生成初稿，不要求填完所有字段。

### Step 2：判断行业/类型

根据方案类型，确定模板中哪些板块需要重点展开：

| 方案类型 | 重点板块 | 说明 |
|----------|----------|------|
| 校园/教育 | 理念体系、空间设计、育人目标 | 参照校园文化模板 |
| 政企/数字化 | 现状诊断、技术架构、实施路径 | 增加技术架构图 |
| 品牌/营销 | 市场分析、策略方案、预算清单 | 强调策略和ROI |
| 活动/展会 | 时间轴、流程设计、资源清单 | 以时间线为主 |
| 通用 | 项目理解、阶段规划、预算汇总 | 使用通用板块 |

### Step 3：参数化模板

读取 `assets/proposal_template.html`，将所有 `{{PLACEHOLDER}}` 替换为实际内容。

**必填替换项（无默认值，必须由用户提供）**：
- `{{PROPOSAL_NAME}}` — 方案完整名称
- `{{PROPOSAL_TYPE}}` — 方案类型标签

**核心替换项**：
- `{{CLIENT_NAME}}` — 客户/受众名称
- `{{BUDGET_RANGE}}` — 预算区间
- `{{PERIOD}}` — 建设/服务周期
- `{{CORE_VALUE}}` — 核心价值主张（一句话）

**板块内容替换项**（按需展开，留空则不显示该板块）：
- `{{SECTION_1}}` ~ `{{SECTION_N}}` — 各阶段/模块内容
- `{{BUDGET_TABLE}}` — 预算明细表（HTML table 格式）
- `{{TIMELINE_ITEMS}}` — 时间线/阶段节点
- `{{TEAM_INTRO}}` — 团队/公司介绍

### Step 4：生成 HTML 文件

将替换后的完整 HTML 写入输出目录，命名格式：

```
{客户名简称}_{方案名简称}_方案.html
```

文件为**单文件纯静态 HTML**，无后端依赖，所有样式和脚本通过 CDN 加载（Tailwind CSS、Mermaid.js、Noto Sans SC）。

### Step 5：向用户呈现

使用 `preview_url` 工具在 IDE 内置浏览器预览，或直接告知用户文件路径。

## 通用板块结构

典型方案包含以下板块，可按实际需要增删：

| # | 板块 | 内容说明 |
|---|------|----------|
| 1 | 项目理解 | 客户需求、现状分析、核心挑战 |
| 2 | 建设路径/总体规划 | Mermaid 流程图展示整体架构 |
| 3 | 分阶段方案详情 | 各阶段目标、工作内容、交付物 |
| 4 | 预算汇总 | 结构化预算表 |
| 5 | 实施建议 | 关键里程碑、风险提示、成功要素 |
| 6 | 下一步行动 | 建议客户接下来的具体动作 |
| 7 | 团队介绍 | 公司背景、核心优势、相关案例 |

## 技术栈

- 纯 HTML + CSS + JS，单文件无后端依赖
- Tailwind CSS（CDN）
- Mermaid.js（CDN，流程图/甘特图渲染）
- Noto Sans SC（Google Fonts，中文支持）

## 输出质量标准

- 预算表合计金额必须与各分项之和一致
- Mermaid 流程图各阶段颜色必须区分
- 手机端可正常阅读（响应式布局）
- 所有价格/人数/面积等数字精确有效
